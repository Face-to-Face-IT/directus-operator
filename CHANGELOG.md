## 0.6.1-alpha - 2025-11-22
### Fixed
* Fixed podAnnotations to properly merge with existing deployment annotations instead of replacing them, preventing loss of annotations added by other control

## 0.6.0-alpha - 2025-11-22
### Added
* Added support for pod annotations

## 0.4.0-alpha - 2025-11-22
### Added
* Added `spec.adminEmail` field to set the initial admin email.
* Added CEL validation rules to ensure valid database configurations.
* Added automatic generation of `KEY`, `SECRET`, and `ADMIN_PASSWORD` in a Kubernetes Secret if not provided.
* Added native SQLite support with optional PersistentVolumeClaim management.
### Changed
* **Breaking**: Removed internal CloudNativePG management. The operator now supports a "Bring Your Own Database" model for greater flexibility.
* **Breaking**: Refactored `spec.database` API. Now supports all Directus-compatible clients (`pg`, `mysql`, `sqlite3`, `mssql`, `oracledb`, `cockroachdb`).

## 0.3.0-alpha - 2025-11-21
### Changed
* Updated container image registry to `ghcr.io/face-to-face-it/directus-operator`.
### Added
* Added support for external databases
* Added support for `npm:` prefix in extensions to install via `npm install`.

## 0.2.0-alpha - 2025-11-21
### Added
* Add support for private NPM registries via .npmrc secret mounting
